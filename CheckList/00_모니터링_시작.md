# 뉴스 사이클 모니터링 시작 가이드

## 사전 준비

### 1. 서버 상태 확인
```bash
# API 서버 헬스 체크
curl -s http://localhost:9501/api/health | python3 -m json.tool

# 매매 상태 확인
curl -s http://localhost:9501/api/trading/status | python3 -m json.tool
```
- `is_trading: true` → 매매 루프 실행 중
- `task_done: false` → 태스크 활성 상태

### 2. 로그 실시간 감시 시작
```bash
# 메인 로그 (전체)
tail -f logs/trading_system.log

# 뉴스 파이프라인만
tail -f logs/trading_system.log | grep -E "\[Step [0-9]|뉴스 파이프라인|news_pipeline"

# 센티넬만
tail -f logs/trading_system.log | grep -E "센티넬|sentinel|anomaly|에스컬레이션"

# Layer 1 + Layer 2 분석만
tail -f logs/trading_system.log | grep -E "Layer [12]|Opus|에이전트|comprehensive"

# 에러만
tail -f logs/trading_system.log | grep -E "ERROR|EXCEPTION|실패|Exception"
```

### 3. Redis 연결 확인
```bash
# Redis CLI 접속
docker exec -it stock_trading-redis-1 redis-cli

# 또는 직접
redis-cli -p 6379

# 연결 테스트
PING
```

### 4. 현재 세션 타입 확인
```bash
curl -s http://localhost:9501/api/trading/status | python3 -c "
import sys, json
d = json.load(sys.stdin)
print(f'세션: {d.get(\"session_type\", \"?\")}')"
```
