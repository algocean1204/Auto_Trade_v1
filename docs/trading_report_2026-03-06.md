## 3. 매매 미실행 근본 원인 분석

### 3-1. 핵심 원인: OBI 게이트 구조적 결함

**진입 전략(EntryStrategy)에 7개 게이트가 순차 평가되는데, 첫 번째 게이트인 OBI(주문흐름편향)에서 모든 종목이 100% 차단되었다.**

```
OBI 게이트 로직:
  order_flow.obi > obi_threshold(0.3) → 통과
  order_flow.obi <= 0.3 → 차단
  order_flow가 None → 통과 + 페널티
```

**문제점**: OrderFlowAggregator가 Redis `order_flow:raw:{ticker}` 키에서 데이터를 읽는데, 이 데이터를 공급하는 WebSocket 피더가 없다. 결과적으로:

1. Redis에 `order_flow:raw:*` 키가 존재하지 않음
2. `_load_flow_data()`가 빈 dict `{"trades": [], "bids": [], "asks": []}` 반환
3. `_calc_obi([], [])` → bid_vol=0, ask_vol=0, total=0 → **OBI=0.0** 반환
4. OrderFlowSnapshot(obi=0.0)이 생성됨 (None이 아님!)
5. OBI 게이트: `0.0 > 0.3` → **False (차단)**

**핵심**: 데이터가 없는 상황인데, 코드는 "데이터가 있고 OBI가 0.0"으로 해석한다. None 체크를 우회하여 항상 차단되는 구조적 결함이다.
