Macro Module Block Design v2.0

Stock Trading AI System V2.0 거시적 모듈 블록 설계서

모듈 = 블록. 각 블록의 IN/OUT은 정확히 1개. 블록을 조립하여 자동매매 파이프라인을 완성한다.
세부사항(보안, 버전 등)은 각 모듈 구현 직전에 별도 설계.

1. 기술 스택

영역기술역할배포
백엔드Python 3.12 FastAPI매매 오케스트레이션, 분석 파이프라인, 모니터링 APIHost (MPS)
AI 엔진Claude Opus/Sonnet Qwen3-30B-A3B MLX종합 분석, 의사결정, 뉴스 분류 (로컬 폴백)API + Host MPS
대시보드Flutter macOS Desktop실시간 모니터링, 매매 제어, 차트Native App
데이터베이스PostgreSQL 17 pgvector거래 내역, 뉴스, 포지션, 벡터 검색Docker
캐시Redis 7실시간 시세, OBI/CVD/VPIN, 세션, Pub/SubDocker
브로커KIS OpenAPI미국 주식 주문, 시세, 잔고 조회외부 API
MPS 필수: MLX 로컬 AI(Qwen3-30B-A3B)는 Apple Silicon MPS 필요. Docker 불가 → Host에서 직접 실행한다.
듀얼 인증: KIS API는 real_auth(시세 조회)와 virtual_auth(모의 매매) 이중 구조이다.

C0

공통 모듈 (Common) — 12개

Feature 폴더 안에 절대 두지 않는다. src/common/에 구현. Manager가 여기서 인스턴스를 꺼내 Atom에 파라미터로 주입(DI).

C0.1

SecretVault — 환경변수 & 키 관리

▶
IN: env_file_path (str) — .env 파일 경로
OUT: SecretProvider (get_secret, get_secret_or_none, has_secret)

.env 파싱, 34개 시크릿 관리 (KIS, DB, AI, Telegram, 외부 API 등), 필수 키 누락 시 fail-fast, 로그 마스킹, 싱글톤. 모든 민감 정보의 유일한 접근 경로.

  • os.environ/os.getenv 직접 사용 금지 (SecretVault만 허용)
  • Gemini 키 로테이션, 키 변경 = .env만 수정
C0.2

DatabaseGateway — PostgreSQL 비동기 세션

▶
IN: database_url (str) — C0.1.get_secret("DATABASE_URL")
OUT: SessionFactory (get_session, close)

SQLAlchemy 2.0 async engine 생성, 세션 풀 관리, async with get_session() as session 패턴.

C0.3

CacheGateway — Redis 비동기 CRUD & Pub/Sub

▶
IN: redis_url (str) — C0.1.get_secret("REDIS_URL")
OUT: CacheClient (read, write, delete, publish, subscribe, aclose)

redis-py 5.0+ async 클라이언트, 연결 풀, pub/sub, TTL 필수.

C0.4

HttpClient — 공유 비동기 HTTP

▶
IN: timeout_config (TimeoutConfig) — 기본 total=30s, connect=10s
OUT: AsyncHttpClient (get, post, close)

aiohttp.ClientSession 공유, 타임아웃, 재시도, 에러 래핑.

C0.5

AiGateway — Claude API + MLX 로컬 통합

▶
IN: anthropic_api_key (C0.1), claude_mode (C0.1), http_client (C0.4)
OUT: AiClient (send_text, send_tools, local_classify)

Claude API 호출 + MLX 로컬 추론 + 폴백 라우팅 (Claude 실패 → MLX). 기존 claude_client.py + fallback_router.py + local_model.py + mlx_classifier.py 통합.

C0.6

BrokerGateway — KIS OpenAPI 통합

▶
IN: kis_app_key, kis_app_secret, kis_real_app_key, kis_real_app_secret, kis_virtual_account, kis_real_account (모두 C0.1), http_client (C0.4)
OUT: BrokerClient (get_price, get_balance, place_order, get_exchange_rate, get_daily_candles)

듀얼 인증 (real_auth + virtual_auth), 토큰 캐시 (1일 1회), 모의거래 시장가→지정가 자동 변환. 기존 kis_auth.py + kis_client.py(1,261줄) 통합 분할.

▶

▶

C0.10

▶
IN: event_type (str), payload (BaseModel)
OUT: EventDeliveryResult (event_type, delivered_to, failed)

Feature 간 직접 호출 금지. 11개 이벤트: ArticleCollected, TradingDecision, PositionChanged, EmergencyLiquidation, CrashDetected, EODStarted, BeastEntry, PyramidTriggered, TiltDetected, InfraHealthChanged, WeeklyReportGenerated.

C0.11

MarketClock — 시간 & 세션 판별

▶
IN: system_clock (Callable) — datetime.now(tz=KST)
OUT: TimeInfo (now_kst, now_et, is_trading_window, session_type, is_regular_session, is_danger_zone, loop_interval_seconds)

is_trading_window(20:00~06:30 KST), 8개 세션 유형(preparation/pre_market/power_open/mid_day/power_hour/final_monitoring/eod_sequence/closed), danger zone 감지(ET 09:30~10:00, 15:30~16:00). 기존 market_hours.py(692줄) + account_safety.py + calendar_helpers.py 통합.

C0.12

TickerRegistry — ETF 유니버스 정보

▶
IN: db_session (SessionFactory) — C0.2
OUT: TickerInfo (get_universe, get_pair, get_exchange_code, get_sector, is_inverse, is_enabled)

ETF 유니버스(SOXL/SOXS, QLD/QID, SSO/SDS, NVDL/NVDS 등), 인버스 페어 매핑, 거래소 코드(NAS/AMS/NYS), 섹터 매핑. 기존 etf_universe.py(888줄) + ticker_mapping.py 통합.

DI 흐름: C0.1(키 로드) → C0.2/3/4/5/6/7(인프라 생성) → Manager(인프라 주입받음) → Atom(파라미터로 전달, 외부 직접 접근 금지)

주문 실행 허용

Beast Mode 4계층 방어망

Network: F6.12 DeadmanSwitch (10s+)
Market: F6.13 MacroFlashCrash (-1%/3min)
Time: F4.3 DangerZone (09:30~10:00, 15:30~16:00 ET)
Price: Beast Hard Stop -1.0%
Fail-Closed: 에러 발생 시 즉시 청산 (홀드 아님). 4계층 중 하나라도 트리거되면 Beast 포지션 즉시 정리.

설계 검증 체크리스트 (15항목)

  • 모든 모듈의 OUT이 정확히 1개인가?
  • SecretVault 외에 os.environ/os.getenv 호출이 없는가?
  • Atom이 30줄 이하인가?
  • Manager가 50줄 이하인가?
  • 파일이 200줄 이하인가?
  • Atom 내부에서 DB/Redis/HTTP를 직접 import하지 않는가?
  • Feature 간 직접 함수 호출 없이 EventBus로만 통신하는가?
  • 모든 IN/OUT이 Pydantic 스키마로 정의되어 있는가?
  • 모든 함수에 타입 힌트가 있는가?
  • 모든 docstring이 한국어로 작성되어 있는가?
  • main.py가 50줄 이하인가?
  • 순환 의존성이 없는가?
  • 매매 윈도우가 20:00~06:30 KST로 올바르게 구현되어 있는가?
  • LaunchAgent 참조가 모두 제거되었는가?
  • 대시보드 버튼 → 시간 체크 → 매매 시작 플로우가 올바른가?
